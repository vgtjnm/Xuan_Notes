  ⭐  初级  （1–8）   单表基础操作
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [✅ ] #01  SELECT 基础
      从 customers 表查询所有客户的 first_name、last_name 和 city，
      只显示来自 'VA' 州的客户。

  [✅ ] #02  ORDER BY
      查询 products 表中所有产品，按 unit_price 从高到低排序，
      只显示前 5 条。

  [✅ ] #03  DISTINCT
      查询 orders 表中出现过的所有不重复的 customer_id，
      并按升序排列。

  [✅ ] #04  WHERE 多条件
      查询 products 表中单价在 1 到 5 之间、且库存大于 20 的
      产品名和价格。

  [ ✅] #05  LIKE 模糊查询
      查询 customers 表中姓氏以字母 'B' 开头的所有客户，
      显示全名和电话。

  [✅ ] #06  IS NULL
      查询 customers 表中 phone 字段为空的客户，
      显示其 id 和 first_name。

  [✅ ] #07  聚合函数
      统计 orders 表中总订单数、最早订单日期和最晚订单日期。

  [✅ ] #08  AS 别名
      查询每个产品的 name 和 unit_price，将价格列命名为「单价(元)」，
      并加一列显示含税价（乘以 1.1）命名为「含税价」。


━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
  ⭐⭐  中级  （9–16）   多表查询
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [ ✅] #09  GROUP BY
      统计每个客户（customer_id）的订单总数，按订单数降序排列。

  [✅ ] #10  JOIN 基础
      查询每笔订单的 order_id、下单日期，以及对应客户的
      first_name 和 last_name。

  [ ✅] #11  多表 JOIN
      列出所有订单明细（order_items），显示产品名称、数量、单价，
      以及所属订单的日期。需要连接 3 张表。

  [ ✅] #12  LEFT JOIN
      查询所有客户及其订单数量，包括从未下过订单的客户（显示为 0）。

  [✅ ] #13  子查询
      找出单价高于所有产品平均单价的产品，显示产品名和价格。

  [ ✅] #14  IN + 子查询
      查询曾经购买过产品编号为 3 的客户的全名，
      使用子查询完成（不允许用 JOIN）。

  [ ✅] #15  字符串函数
      将 customers 表中每个客户的 first_name 和 last_name 拼接为
      「姓名」列（格式：Last, First），并转换为大写。

  [✅ ] #16  日期函数
      查询 orders 表中 2019 年下的所有订单，
      显示订单号、下单日期和距今天数。

  ⭐⭐⭐  高级  （17–24）   复杂业务场景
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

  [ ] #17  复杂聚合
      计算每个客户的总消费金额（quantity × unit_price），显示客户姓名
      和总金额，按总金额降序，只取前 3 名。

  [ ] #18  CASE WHEN
      给 products 表的每个产品根据价格打标签：
      低于 20 = 「经济」，20–50 = 「标准」，高于 50 = 「高端」，
      显示产品名、价格和标签列。

  [ ] #19  自连接
      找出与客户 'Babara MacCaffery' 居住在同一城市的其他客户，
      显示他们的全名和城市。

  [ ] #20  EXISTS
      找出至少下过一笔总金额超过 100 元订单的客户
      （金额 = quantity × unit_price 之和），用 EXISTS 实现。

  [ ] #21  视图 VIEW
      创建名为 vw_customer_orders 的视图，包含每个客户的 id、全名、
      订单总数和总消费额。然后查询该视图，找出总消费额最高的客户。

  [ ] #22  存储过程
      创建存储过程 get_customer_orders，接收 customer_id 参数，
      返回该客户的所有订单详情（含产品名、数量、小计）。
      然后调用它查询 id = 1 的客户。

  [ ] #23  事务处理
      模拟一次「库存扣减 + 订单新增」的原子操作：
      向 orders 插入新订单，向 order_items 插入对应明细，
      并将 products 中对应产品的 quantity_in_stock 减少相应数量。
      三步操作包在一个事务里，若出错则回滚。

  [ ] #24  综合挑战 🏆
      生成一份「客户价值报告」：统计每位客户的总订单数、总消费额、
      平均每单金额，以及他们最近一次下单距今天数。
      按总消费额降序展示全部结果，用一条 SQL 完成。
